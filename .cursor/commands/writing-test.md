# Viết Test

Hỗ trợ viết tests để đạt 100% coverage.

## Bước 1: Thu thập thông tin
Hỏi user về:
- Tên feature và tóm tắt thay đổi
- Môi trường đích (backend, frontend, full-stack)
- Bộ test hiện có

## Bước 2: Tạo tests
Với mỗi module/function:
- Liệt kê scenarios (happy path, edge cases, xử lý lỗi)
- Tạo test cases với assertions
- Tham khảo utilities/mocks hiện có
- Cung cấp test snippets

## Bước 3: Kiểm tra coverage
- Đề xuất lệnh tooling để kiểm tra coverage
- Xác định files/functions cần thêm tests
- Đề xuất tests bổ sung nếu coverage < 100%

---

**Bắt đầu:** Hỏi tên feature và tóm tắt thay đổi.
